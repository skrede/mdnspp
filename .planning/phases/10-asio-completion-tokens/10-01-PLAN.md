<objective>
Rename all four public type entry-point methods to async_ prefix and add callback-based completion handler support. This is the foundational API transformation — `discover()` becomes `async_discover(type, callback)`, `query()` becomes `async_query(name, qtype, callback)`, `start()` becomes `async_observe(callback)` (observer) and `async_start(callback)` (service_server). Old methods are removed (clean break for v2.0). Completion callbacks are stored and fired at the natural completion point (silence timeout for query-style, stop() for long-lived).

Purpose: Establish the async_ naming convention and callback completion path (API-09 callback entry point, API-10 callback fires with correct results). This works for both MockPolicy and NativePolicy without any ASIO dependency. Plan 10-02 layers ASIO completion token template support on top.
Output: All four public types have async_ methods accepting callback completions; all tests and examples updated.
</objective>

  <name>Task 1: Transform query-style types (service_discovery, querent) to async_ API with callback completion</name>
  <files>include/mdnspp/service_discovery.h, include/mdnspp/querent.h</files>
  <action>
**service_discovery.h:**

1. Add a completion callback type alias:
   ```cpp
   using completion_handler = std::function<void(std::error_code, std::vector<mdns_record_variant>)>;
   ```

2. Add a private member to store the completion handler:
   ```cpp
   completion_handler m_on_completion;  // fires once at silence timeout
   ```

3. Rename `discover(std::string_view service_type)` to `async_discover(std::string_view service_type, completion_handler on_done)`. The method body is the same as `discover()` but:
   - Store `on_done` into `m_on_completion` at the top
   - Change the `on_silence` lambda (currently `[this]() { m_loop->stop(); }`) to:
     ```cpp
     [this]() {
         m_loop->stop();
         if (auto h = std::exchange(m_on_completion, nullptr); h)
             h(std::error_code{}, std::move(m_results));
     }
     ```
   - The `on_done` parameter is decay-copied by `std::function` — safe for deferred use later in Plan 02.

4. Remove the old `discover()` method entirely (clean break per CONTEXT.md).

5. Update `stop()` to also fire the completion handler (for early termination):
   ```cpp
   void stop() {
       if (m_loop) {
           m_loop->stop();
           if (auto h = std::exchange(m_on_completion, nullptr); h)
               h(std::error_code{}, std::move(m_results));
       }
   }
   ```

**querent.h:**

Apply the identical transformation:
1. Add `completion_handler` type alias (same signature as service_discovery).
2. Add `m_on_completion` private member.
3. Rename `query(std::string_view name, uint16_t qtype)` to `async_query(std::string_view name, uint16_t qtype, completion_handler on_done)`.
4. Store `on_done`, update `on_silence` lambda to fire the handler with results.
5. Remove old `query()` method.
6. Update `stop()` to fire the completion handler on early termination.

**Important implementation details:**
- `results()` accessor is KEPT (per CONTEXT.md: "results accessible both via completion token AND via accessor after completion"). The completion handler receives `std::move(m_results)`, so after completion the accessor returns an empty vector. This is acceptable — the user got results via the handler. If users want both, they should copy from results() before completion fires.
- Actually, better: fire the handler with a COPY of m_results (not move), so results() remains populated. The vector is small (mDNS responses are tiny). Use: `h(std::error_code{}, m_results);` (copy, not move).

<task type="auto">
  <name>Task 2: Transform long-lived types (observer, service_server) to async_ API with callback completion</name>
  <files>include/mdnspp/observer.h, include/mdnspp/service_server.h</files>
  <action>
**observer.h:**

1. Add a completion callback type alias:
   ```cpp
   using completion_handler = std::function<void(std::error_code)>;
   ```

2. Add a private member:
   ```cpp
   completion_handler m_on_completion;  // fires when stop() is called
   ```

3. Rename `start()` to `async_observe(completion_handler on_done = {})`. The method body is the same as `start()` but:
   - Store `on_done` into `m_on_completion` at the top.
   - The rest of the body (creating recv_loop, calling m_loop->start()) is unchanged.
   - The `on_done` parameter defaults to `{}` (empty function) so `async_observe()` with no args still works for NativePolicy users who just want to observe and call stop() manually.

4. Remove old `start()` method.

5. Update `stop()` to fire the completion handler:
   ```cpp
   void stop() {
       if (m_stopped.exchange(true, std::memory_order_acq_rel))
           return;
       if (auto h = std::exchange(m_on_completion, nullptr); h)
           h(std::error_code{});
   }
   ```
   Note: `stop()` fires the handler BEFORE destroying the loop (loop is destroyed in destructor, not in stop() — this is the existing design and must be preserved for callback-safe stop).

**service_server.h:**

1. Add a completion callback type alias:
   ```cpp
   using completion_handler = std::function<void(std::error_code)>;
   ```

2. Add a private member:
   ```cpp
   completion_handler m_on_completion;  // fires when stop() is called
   ```

3. Rename `start()` to `async_start(completion_handler on_done = {})`. The method body is the same as `start()` but stores `on_done` into `m_on_completion`.

4. Remove old `start()` method.

5. Update `stop()` to fire the completion handler:
   ```cpp
   void stop() {
       if (m_stopped.exchange(true, std::memory_order_acq_rel))
           return;
       if (auto h = std::exchange(m_on_completion, nullptr); h)
           h(std::error_code{});
       m_response_timer.cancel();
       m_loop.reset();
   }
   ```
   Fire the handler BEFORE canceling the timer and destroying the loop, so the handler can safely access the server state.

**Important:** The `on_silence` lambda for observer and service_server is `[](){}` (no-op) — these are long-lived operations that run until `stop()`. The completion fires from `stop()`, not from silence. This is the CONTEXT.md locked decision.
