  <action>
Add template `async_` overloads guarded by `#ifdef ASIO_STANDALONE` to each public type header. These overloads accept any ASIO completion token and use `asio::async_initiate` to dispatch. The non-template callback overloads from Plan 10-01 remain as-is for NativePolicy users.

**Include guard strategy:** At the top of each public type header, after existing includes, add:
```cpp
#ifdef ASIO_STANDALONE
#include <asio/async_result.hpp>
#include <asio/dispatch.hpp>
#include <asio/executor_work_guard.hpp>
#include <asio/bind_allocator.hpp>
#include <asio/recycling_allocator.hpp>
#include <asio/associated_allocator.hpp>
#endif
```
This ensures ASIO headers are only included when ASIO is available (ASIO_STANDALONE is defined by standalone ASIO; it is set automatically when `#include <asio.hpp>` has been included, or when building with `mdnspp_asio` target which defines it).

**service_discovery.h — add template overload:**

Inside the class, add (after the non-template `async_discover`):

```cpp
#ifdef ASIO_STANDALONE
    /// ASIO completion token overload — accepts use_future, use_awaitable, deferred, or any callable.
    template <asio::completion_token_for<void(std::error_code, std::vector<mdns_record_variant>)>
                  CompletionToken>
    auto async_discover(std::string_view service_type, CompletionToken&& token)
    {
        return asio::async_initiate<
            CompletionToken,
            void(std::error_code, std::vector<mdns_record_variant>)>(
            [this](auto handler, std::string svc_type)
            {
                auto work = asio::make_work_guard(handler);

                // Store handler + work for later dispatch.
                // Type-erase into std::function + executor_work_guard<any_io_executor>.
                auto ex = work.get_executor();
                m_on_completion = [h = std::move(handler), w = std::move(work)](
                    std::error_code ec, std::vector<mdns_record_variant> results) mutable
                {
                    auto alloc = asio::get_associated_allocator(
                        h, asio::recycling_allocator<void>());
                    asio::dispatch(w.get_executor(),
                        asio::bind_allocator(alloc,
                            [h2 = std::move(h), ec, r = std::move(results)]() mutable
                            {
                                std::move(h2)(ec, std::move(r));
                            }));
                };

                // Run the discover logic (same as non-template version body after storing handler)
                assert(m_loop == nullptr);
                m_results.clear();
                m_service_type = std::move(svc_type);
                if (!m_service_type.empty() && m_service_type.back() == '.')
                    m_service_type.pop_back();

                auto query_bytes = detail::build_dns_query(m_service_type, 12);
                m_socket.send(endpoint{"224.0.0.251", 5353},
                              std::span<const std::byte>(query_bytes));

                m_loop = std::make_unique<recv_loop<P>>(
                    m_socket, m_timer, m_silence_timeout,
                    [this](std::span<std::byte> data, endpoint sender) -> bool
                    {
                        // ... same on_packet lambda as existing ...
                        std::vector<mdns_record_variant> batch;
                        detail::walk_dns_frame(
                            std::span<const std::byte>(data.data(), data.size()),
                            sender,
                            [&batch](mdns_record_variant rec) { batch.push_back(std::move(rec)); });
                        bool relevant = std::any_of(batch.begin(), batch.end(),
                            [this](const mdns_record_variant &rec) {
                                return std::visit([this](const auto &r) { return r.name == m_service_type; }, rec);
                            });
                        if (relevant) {
                            if (m_on_record)
                                for (const auto &rec : batch) m_on_record(rec, sender);
                            m_results.insert(m_results.end(),
                                std::make_move_iterator(batch.begin()),
                                std::make_move_iterator(batch.end()));
                        }
                        return relevant;
                    },
                    [this]() {
                        m_loop->stop();
                        if (auto h = std::exchange(m_on_completion, nullptr); h)
                            h(std::error_code{}, m_results);
                    });
                m_loop->start();
            },
            token,
            std::string(service_type));  // decay-copy for deferred safety
    }
#endif
```

**IMPORTANT:** The template overload's initiation lambda contains the FULL discover logic (not delegating to the non-template version) because `async_initiate` with `deferred` must not execute the logic eagerly — the initiation lambda IS the deferred body. The non-template `async_discover(type, callback)` remains separate for NativePolicy users.

**To avoid code duplication:** Extract the common discover body into a private `do_discover(std::string svc_type)` method that sets up the recv_loop and starts it. Both the non-template `async_discover` and the template overload call `do_discover` inside their respective initiation paths. The `do_discover` method:
- Takes `std::string` (already decay-copied)
- Assumes `m_on_completion` is already set
- Sets up m_service_type, sends query, creates recv_loop with on_silence that fires m_on_completion, calls m_loop->start()

This eliminates duplication and keeps the template method small (just the work guard + handler storage + call to do_discover).

**querent.h — identical pattern:**
- Template `async_query(string_view, uint16_t, CompletionToken&&)` using `async_initiate`
- Extract common body into `do_query(std::string name, uint16_t qtype)`
- Decay-copy both name (string) and qtype (trivial copy) as args to `async_initiate`

**observer.h — long-lived pattern:**
- Template `async_observe(CompletionToken&&)` with signature `void(std::error_code)`
- Initiation lambda stores handler + work guard in `m_on_completion`
- `stop()` fires the handler via the stored completion (already dispatch-wrapped)
- Extract common body into `do_observe()`

**service_server.h — long-lived pattern:**
- Template `async_start(CompletionToken&&)` with signature `void(std::error_code)`
- Same pattern as observer
- Extract common body into `do_start()`

**Key implementation notes from RESEARCH.md:**
1. Always `asio::make_work_guard(handler)` — prevents premature `io_context::run()` return
2. Always `asio::dispatch(work.get_executor(), ...)` — never call handler directly (TSan safety)
3. Always `asio::get_associated_allocator` + `asio::bind_allocator` — respects handler's allocator
4. Use `std::exchange(m_on_completion, nullptr)` when firing — prevents double-fire
5. decay-copy `string_view` to `std::string` before passing to `async_initiate` — deferred safety (CONTEXT.md locked decision)

SCENARIO("async_observe with use_awaitable suspends until stop", "[completion_token][use_awaitable]") {
    asio::io_context io;
    try {
        auto obs = std::make_shared<mdnspp::observer<mdnspp::AsioPolicy>>(io,
            [](mdnspp::mdns_record_variant, mdnspp::endpoint){});

        // Schedule a stop after the co_await arms
        asio::steady_timer stop_timer{io, std::chrono::milliseconds(100)};
        stop_timer.async_wait([obs](auto) { obs->stop(); });

        bool completed = false;
        asio::co_spawn(io,
            [obs, &completed]() -> asio::awaitable<void> {
                co_await obs->async_observe(asio::use_awaitable);
                completed = true;
            },
            asio::detached);

        io.run();
        REQUIRE(completed);

SCENARIO("async_discover with deferred does not initiate I/O until launched", "[completion_token][deferred]") {
    asio::io_context io;
    try {
        mdnspp::service_discovery<mdnspp::AsioPolicy> sd{io, std::chrono::milliseconds(500)};

        // Create deferred operation — should NOT send any packets yet

        // io_context should have no work yet (deferred hasn't launched)
        // Launch the operation with a callback
