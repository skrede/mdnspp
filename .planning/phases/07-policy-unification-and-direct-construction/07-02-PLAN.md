```cpp
namespace mdnspp {
template <typename S>
concept SocketLike = requires(S &s, endpoint ep, std::span<const std::byte> send_data,
    std::function<void(std::span<std::byte>, endpoint)> handler) {
    { s.async_receive(handler) } -> std::same_as<void>;
    { s.send(ep, send_data) } -> std::same_as<void>;
    { s.close() } -> std::same_as<void>;
};

template <typename T>
concept TimerLike = requires(T &t, std::chrono::milliseconds dur,
    std::function<void(std::error_code)> handler) {
    t.expires_after(dur);
    { t.async_wait(handler) } -> std::same_as<void>;
    { t.cancel() } -> std::same_as<void>;
};

template <typename P>
concept Policy = /* SocketLike + TimerLike + constructible_from constraints */;
}
```

Migrate all four public types from `template <SocketPolicy S, TimerPolicy T>` to `template <Policy P>`. Replace `create()` factories with direct public constructors. For each type, follow the same transformation pattern:

---

**observer.h:**

1. Replace includes: `socket_policy.h` and `timer_policy.h` with `policy.h`. Remove `<expected>` include (no more create()). Remove `mdns_error.h` include (no more mdns_error return).

2. Change template from `template <SocketPolicy S, TimerPolicy T>` to `template <Policy P>`.

3. Change member types from `S m_socket; T m_timer;` to `typename P::socket_type m_socket; typename P::timer_type m_timer;`.

4. Change `std::unique_ptr<recv_loop<S, T>>` to `std::unique_ptr<recv_loop<P>>`.

5. **DELETE the `create()` factory entirely.** Make the constructor public.

6. **Throwing constructor** (public):
   ```cpp
   explicit observer(typename P::executor_type ex, record_callback callback)
       : m_socket(ex)
       , m_timer(ex)
       , m_callback(std::move(callback))
       , m_loop(nullptr)
       , m_stopped(false)
   {}
   ```

7. **Non-throwing constructor** (public):
   ```cpp
   observer(typename P::executor_type ex, record_callback callback, std::error_code &ec)
       : m_socket(ex, ec)
       , m_timer(ex)
       , m_callback(std::move(callback))
       , m_loop(nullptr)
       , m_stopped(false)
   {}
   ```

8. **socket() accessor**: Change from `m_loop ? m_loop->socket() : m_socket` to simply `return m_socket;` — observer owns the socket directly now, recv_loop only borrows a reference.

9. **Add timer() accessor** (const and non-const, same as socket) returning `m_timer`.

10. **Move constructor**: Update types from `S`/`T` to `P::socket_type`/`P::timer_type`.

11. **start()**: Change `recv_loop<S, T>` to `recv_loop<P>`. The constructor args remain `m_socket, m_timer, ...` — recv_loop now borrows by reference.

---

**service_discovery.h:**

Same pattern as observer, with these specifics:

1. Constructor takes: `(executor_type ex, std::chrono::milliseconds silence_timeout, record_callback on_record = {})`
   - Silence timeout is a constructor parameter per CONTEXT.md
   - on_record callback is optional (defaulted)

2. Non-throwing overload: `(executor_type ex, std::chrono::milliseconds silence_timeout, record_callback on_record, std::error_code &ec)` — ec is LAST. Also provide an overload without on_record: `(executor_type ex, std::chrono::milliseconds silence_timeout, std::error_code &ec)`.

3. **socket() accessor**: Simply returns `m_socket` (no more `m_loop ? m_loop->socket() : m_socket` delegation).

4. **Add timer() accessor** returning `m_timer`.

5. **discover()**: Change `recv_loop<S, T>` to `recv_loop<P>`. Socket send uses `m_socket.send(...)` directly (not through loop).

6. The `m_loop->stop()` in the on_silence callback stays the same (stops the recv_loop).

---

**querent.h:**

Same pattern as service_discovery:

1. Constructor: `(executor_type ex, std::chrono::milliseconds silence_timeout, record_callback on_record = {})`
2. Non-throwing overloads with ec last.
3. socket()/timer() accessors return members directly.
4. query(): Change `recv_loop<S, T>` to `recv_loop<P>`.

---

**service_server.h:**

This is the most complex due to dual timers:

1. Members change to:
   ```cpp
   typename P::socket_type m_socket;
   typename P::timer_type m_response_timer;
   typename P::timer_type m_recv_timer;
   ```

2. **Throwing constructor:**
   ```cpp
   explicit service_server(typename P::executor_type ex, service_info info,
                           query_callback on_query = {})
       : m_socket(ex)
       , m_response_timer(ex)
       , m_recv_timer(ex)
       , m_info(std::move(info))
       , m_on_query(std::move(on_query))
       , m_rng(std::random_device{}())
       , m_loop(nullptr)
       , m_stopped(false)
   {}
   ```

3. **Non-throwing constructor:**
   ```cpp
   service_server(typename P::executor_type ex, service_info info,
                  query_callback on_query, std::error_code &ec)
       : m_socket(ex, ec)
       , m_response_timer(ex)
       , m_recv_timer(ex)
       , m_info(std::move(info))
       , m_on_query(std::move(on_query))
       , m_rng(std::random_device{}())
       , m_loop(nullptr)
       , m_stopped(false)
   {}
   ```
   Also add overload without on_query: `(executor_type ex, service_info info, std::error_code &ec)`.

4. **socket() accessor**: Simply returns `m_socket` (no delegation to m_loop).

5. **timer() accessor**: Returns `m_response_timer` (same name, same semantics as current).

6. **recv_timer() accessor**: NEW — returns `m_recv_timer` (per CONTEXT.md: named accessors for service_server's two timers).

7. **send_response()**: Change `m_loop->socket().send(...)` to `m_socket.send(...)` — this is the critical ownership inversion. The socket is owned by service_server, not by recv_loop. Remove the `if (m_loop)` guard — m_socket is always valid.

8. **start()**: Change `recv_loop<S, T>` to `recv_loop<P>`. Pass `m_socket` and `m_recv_timer` to recv_loop constructor (by reference — recv_loop borrows).

9. **Move constructor/assignment**: Update types from `S`/`T` to `P::socket_type`/`P::timer_type`.
