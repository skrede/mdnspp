From include/mdnspp/socket_policy.h:
```cpp
template <typename S>
concept SocketPolicy = requires(S &s, endpoint ep, std::span<const std::byte> send_data, std::function<void(std::span<std::byte>, endpoint)> handler)
{
    { s.async_receive(handler) } -> std::same_as<void>;
    { s.send(ep, send_data) } -> std::same_as<void>;
    { s.close() } -> std::same_as<void>;
};
```

From include/mdnspp/timer_policy.h:
```cpp
template <typename T>
concept TimerPolicy = requires(T &t, std::chrono::milliseconds dur, std::function<void(std::error_code)> handler)
{
    t.expires_after(dur);
    { t.async_wait(handler) } -> std::same_as<void>;
    { t.cancel() } -> std::same_as<void>;
};
```

**1. Create `include/mdnspp/policy.h`:**

Define three concepts in namespace `mdnspp`:

- `SocketLike<S>`: Exact same requires-expression as current `SocketPolicy` concept in `socket_policy.h`. This is a rename/adaptation. Include `mdnspp/endpoint.h` for the `endpoint` type.

- `TimerLike<T>`: Exact same requires-expression as current `TimerPolicy` concept in `timer_policy.h`. This is a rename/adaptation.

- `Policy<P>`: Compound concept requiring:
  - `typename P::executor_type` exists
  - `typename P::socket_type` exists
  - `typename P::timer_type` exists
  - `SocketLike<typename P::socket_type>`
  - `TimerLike<typename P::timer_type>`
  - `std::constructible_from<typename P::socket_type, typename P::executor_type>`
  - `std::constructible_from<typename P::timer_type, typename P::executor_type>`

Also add a non-throwing constructor constraint for both socket and timer:
  - `std::constructible_from<typename P::socket_type, typename P::executor_type, std::error_code &>`

Keep the `detail::always_false` helper in this header (moved from socket_policy.h) since policy.h will be the single concept header going forward.

Use header guard `HPP_GUARD_MDNSPP_POLICY_H`.
