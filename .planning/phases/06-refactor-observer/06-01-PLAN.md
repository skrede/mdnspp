<feature>
  <name>observer<S, T> class template with perpetual receive and callback delivery</name>
  <files>include/mdnspp/observer.h, tests/observer_test.cpp, tests/CMakeLists.txt</files>
  <behavior>
    - SCENARIO: packet delivery — When observer is started with enqueued packets, each DNS record from each packet is delivered to the callback with the correct sender endpoint
    - SCENARIO: multiple packets — When multiple packets are enqueued, observer delivers records from all of them before stopping
    - SCENARIO: stop idempotency — When stop() is called twice, the second call is a no-op with no crash or assertion failure
    - SCENARIO: start and stop lifecycle — observer can be created, started, and stopped without packet delivery (empty queue)
    - SCENARIO: callback-safe stop — When stop() is called from within the record callback, observer does not deadlock or crash (the stop flag is set, no further records are delivered)
    - SCENARIO: malformed packet — When a malformed (truncated) packet is enqueued, observer skips it without crashing and delivers no records for that packet
  </behavior>
  <implementation>
    **observer.h rewrite** — Replace the current stub with a full `<S, T>` class template:

    1. Template params: `SocketPolicy S, TimerPolicy T` (same as service_server)
    2. Callback type: `using record_callback = std::function<void(mdns_record_variant, endpoint)>`
    3. Factory: `[[nodiscard]] static std::expected<observer, mdns_error> create(S socket, T timer, record_callback callback)` — takes ONE timer (unlike service_server's two). Private constructor moves all three args into members.
    4. `start()` — creates `std::unique_ptr<recv_loop<S, T>>` from moved `m_socket` and `m_timer`, with silence timeout `std::chrono::hours(24 * 365)` (run-until-stop). Packet handler is `[this](std::span<std::byte> data, endpoint sender) { on_packet(data, sender); }`. Silence callback is no-op. Calls `m_loop->start()`. Assert `m_loop == nullptr` to prevent double-start.
    5. `stop()` — sets `m_stopped.exchange(true, std::memory_order_acq_rel)` and returns if already true (idempotent). Does NOT call `m_loop.reset()` here — this is critical for callback-safe stop (Pitfall 5 from RESEARCH). recv_loop will not re-arm after m_stopped is set because recv_loop::stop() is called by ~recv_loop which runs in ~observer.
    6. `~observer()` — stores `m_stopped = true` then calls `m_loop.reset()` (safe — destructor is never called from within callback chain).
    7. `on_packet(std::span<std::byte> data, endpoint sender)` — checks `m_stopped` flag; if stopped, return immediately. Calls `detail::walk_dns_frame(std::span<const std::byte>(data.data(), data.size()), sender, [this, sender](mdns_record_variant rec) { if (!m_stopped.load(...)) m_callback(std::move(rec), sender); })`. The inner lambda captures sender by value and passes it alongside the record.
    8. Non-copyable, non-movable (delete all four: copy ctor, copy assign, move ctor, move assign). Simpler than service_server per RESEARCH open question 2.
    9. `socket()` const/non-const test accessor: returns `m_loop->socket()` if loop exists, else `m_socket`.

    Members: `S m_socket`, `T m_timer`, `record_callback m_callback`, `std::unique_ptr<recv_loop<S,T>> m_loop`, `std::atomic<bool> m_stopped`.

    Includes: `socket_policy.h`, `timer_policy.h`, `records.h`, `mdns_error.h`, `endpoint.h`, `recv_loop.h`, `dns_wire.h`, `<expected>`, `<memory>`, `<atomic>`, `<functional>`, `<span>`, `<cassert>`.

    **tests/observer_test.cpp** — BDD-style Catch2 tests:
    - Copy byte-building helpers (bytes(), push_u16_be, push_u32_be, encode_name, make_ptr_response) from service_discovery_test.cpp.
    - 6 scenarios per behavior list above.
    - Test pattern: Create MockSocketPolicy, enqueue packets, create observer via `observer<MockSocketPolicy, MockTimerPolicy>::create(...)`, call `start()`, assert callback invocations, call `stop()`.

    **tests/CMakeLists.txt** — Add `make_test(observer_test)` after the existing `make_test(service_server_test)` line.
  </implementation>
</feature>
